# 命令行编译 题解

## 题意概述

给定若干条合法的 `g++` 命令行，需要解析出：

1. 源文件
2. `C++` 标准
3. 优化等级
4. 警告等级列表
5. 输出文件

多个 token 之间可能有任意个空格，命令行开头和结尾也可能有空格。

## 正解思路

逐行读取完整字符串，然后用 `stringstream` 按空白切分 token。

- 第一个 token 固定是 `g++`，可以直接跳过。
- 不以 `-` 开头的 token 就是唯一的源文件。
- `-std=` 开头的 token 记录标准；若不存在，默认 `c++17`。
- `-O` 开头的 token 记录优化等级；若不存在，输出 `None`。
- `-W` 开头的 token 依照输入顺序收集；若不存在，输出 `None`。
- `-o` 开头的 token 记录输出文件；若不存在，默认 `a.out`。

## 正确性说明

题目已经保证输入命令合法，因此每个 token 都只会属于上述某一种语义。

- 唯一不带前导 `-` 的 token 必然是源文件。
- 最多一个 `-std=`、最多一个 `-O*`、最多一个 `-o*`，因此覆盖记录不会产生歧义。
- 所有 `-W*` token 都需要保留并按出现顺序输出，顺序扫描即可满足。

所以该解析过程与题意定义完全一致，得到的五行结果一定正确。

## 复杂度分析

设单条命令长度为 `L`。

- 时间复杂度：`O(L)`
- 空间复杂度：`O(L)`

## 易错点

- 必须整行读取，不能直接用 `cin >>` 按 token 读完整个测试文件。
- 默认标准是 `c++17`，不是题面里示例使用的其他标准。
- 默认输出文件是 `a.out`。
- 多个警告等级要保持输入顺序。
