# 命令行编译 题解

这是一道直接模拟题。

## 思路

每一条命令本质上只是一个由空格分隔的 token 序列。由于相邻 token 之间可能有任意多个空格，最自然的做法是先把整行按空白切成 token，再逐个分类：

- `g++` 一定在开头，可以直接跳过
- 不以 `-` 开头的 token 就是源文件
- `-std=` 开头的 token 记录标准
- `-O` 开头的 token 记录优化等级
- `-W` 开头的 token 按顺序加入警告列表
- `-o` 开头的 token 记录输出文件

最后把缺失项补成默认值即可。

## 正确性说明

题目保证每条命令合法，并且只会出现题面列出的这几类 token。

因此，除了开头固定的 `g++` 之外，任意一个 token 必然且只会属于下面某一类：

- 源文件
- 标准
- 优化等级
- 警告等级
- 输出文件

我们的分类规则与题面对这些 token 的定义完全一致，所以每个 token 都会被放进正确的位置。

同时：

- 如果某一类 token 存在，我们就会在扫描到它时记录它的真实值
- 如果某一类 token 不存在，我们就输出题目规定的默认值

因此最终输出恰好就是题目要求的五项信息。

## 实现细节

- C++ 中直接用 `stringstream` 或手写跳空格切词都可以。
- 输出优化等级和警告等级时，要去掉最前面的横杠 `-`，所以：
  - `-O2` 应输出 `O2`
  - `-Wall` 应输出 `Wall`
- 输出文件 `-ofoo.out` 需要去掉前缀 `-o`，保留后面的 `foo.out`
- 多个警告必须保持输入顺序，不能排序

## 复杂度分析

设所有命令总长度为 $L$。

- 时间复杂度：$O(L)$
- 空间复杂度：$O(L)$（如果按行切词），或 $O(1)$ 额外空间（如果边扫边解析）
