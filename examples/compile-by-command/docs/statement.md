# 命令行编译

## 题目内容

在 ICPC 比赛中，选手通常会直接用命令行调用 `g++` 编译程序。现在老师给你若干条已经保证合法的编译命令，请你把其中关心的几类 token 解析出来。

在本题中，一条命令只会由下面这些类型的 token 组成：

- 编译器：第一段 token，固定为 `g++`
- 源文件：一个**不以 `-` 开头**的 token，且保证**有且仅有一个**
- `C++` 标准：一个以 `-std=` 开头的 token
- 优化等级：一个以 `-O` 开头的 token
- 警告等级：若干个以 `-W` 开头的 token，且互不相同
- 输出文件：一个以 `-o` 开头的 token；注意本题中输出文件总是写成 `-ofoo.out` 这种形式，`-o` 与文件名之间**没有空格**

例如：

```text
g++ -std=c++20 -O2 -Wall foo.cc -ofoo.out
```

表示它会：

- 编译源文件 `foo.cc`
- 使用标准 `c++20`
- 开启优化 `O2`
- 开启警告 `Wall`
- 把输出文件设为 `foo.out`

如果某个可选信息没有显式给出，则按下面的默认值处理：

- `C++` 标准默认是 `c++17`
- 优化等级默认是 `None`
- 警告等级默认是 `None`
- 输出文件默认是 `a.out`

## 输入格式

**本题有多组数据。**

第一行一个正整数 $T\ (1 \le T \le 2000)$，表示有 $T$ 条编译命令。

接下来 $T$ 行，每行一条长度不超过 $200$ 的字符串，表示一条编译命令。

这些命令保证合法。具体来说：

- 第一段 token 一定是 `g++`
- 源文件 token 有且仅有一个
- `-std=`、`-O`、`-o` 这三类 token 各自至多出现一次
- 不会出现两个完全相同的警告 token
- `-std=`、`-O`、`-W`、`-o` 后面都不会是空串
- 每条命令只会使用题面列出的这些 token 类型

请注意：

- token 之间用空格分隔
- 相邻两个 token 之间可以有任意多个空格
- 整条命令的开头和结尾也可能有空格
- 本题中所有字符都属于可打印 ASCII 字符

## 输出格式

对于每一条编译命令，输出五行，依次表示：

1. 源文件
2. `C++` 标准
3. 优化等级
4. 警告等级
5. 输出文件

具体要求如下：

- 标准：如果未指定，输出 `c++17`
- 优化等级：输出时保留前导大写字母 `O`，但不要带横杠 `-`；如果未指定，输出 `None`
- 警告等级：输出时保留前导大写字母 `W`，但不要带横杠 `-`
  如果存在多个警告，按**输入中出现的顺序**输出，相邻两个之间用一个空格分隔
  如果没有警告，输出 `None`
- 输出文件：输出时不要带前导 `-o`；如果未指定，输出 `a.out`

## 样例

```input1
3
g++ foo.cc
g++ -O2 -Wall -Wextra -std=c++11 foo.cc -owoshinailong.out
g++ qaq.cpp -oqwq.exe -Ofast -std=gnu++23
```

```output1
foo.cc
c++17
None
None
a.out
foo.cc
c++11
O2
Wall Wextra
woshinailong.out
qaq.cpp
gnu++23
Ofast
None
qwq.exe
```

## 样例说明

第二条命令中：

- 源文件是 `foo.cc`
- 标准是 `c++11`
- 优化等级是 `O2`
- 警告按照输入顺序依次是 `Wall`、`Wextra`
- 输出文件是 `woshinailong.out`

注意本题要求的是**解析**，不是模拟真实编译行为。
